# AGENTS.md

Guidance for AI coding agents working in this repository. Human contributors should start
from `README.md`.

## What this package is

`@adsq/angular-signal-store` — an Angular reactive store built on a callable nested proxy.
Reading a path (`store.user.name()`) subscribes the caller to that exact path; assigning to
it (`store.user.name = 'Ada'`) wakes only that path's consumers.

**The full API reference for agents lives in [`SKILL.md`](./SKILL.md)** — read it before
writing code that uses the store. It is written in the [Agent Skills](https://agentskills.io)
format and covers the architecture rule, the template rules, batching, wake modes, and
JSNQ. Consumers of the published package can install it as a skill; see the README section
"Use With AI Coding Agents".

## Repository layout

- `src/core/` — `SignalStore` service, `CreateStore`, named registry, devtools contract.
- `src/proxy/` — the callable nested proxy handlers.
- `src/operations/`, `src/utils/` — array operations, path utilities, wake scheduling.
- `src/devtools/` — optional adapter, exposed as the `/devtools` secondary entry point.
- `test/` — headless smoke and throughput bench (run under `bun`, no browser needed).

This package is the **library only**. Demo components, benches, and app-level specs live in
the main application repository, not here.

## Working rules

- **`dist/` is generated by ng-packagr and git-ignored.** Never edit or commit it;
  `prepack` builds it for npm.
- **The core must never statically import `@adsq/jsnq/core/pipeline` or
  `core/pipeline-fastpath`.** They belong to the `/jsnq` entry point and reach the core
  only through `src/core/jsnq-contract.ts`. A static import silently adds ~12 kB gzip to
  every application, including those that never query. `core/data-engine` is fine — it is
  the small path engine the core legitimately shares.
- **`sideEffects` must keep matching the jsnq entry file** (`**/*-jsnq.mjs`). ng-packagr
  copies the field verbatim into `dist/package.json`, so a `./dist/...`-prefixed pattern
  silently stops matching there and bundlers drop the registration import.
- **Do not reach into `@adsq/jsnq` internals.** Use its documented entries only. The engine
  is a separate package — do not vendor or fork it here.
- Angular, RxJS, and JSNQ are **peer dependencies**. Do not promote them to regular
  dependencies; that would pull a second framework runtime into consuming apps.
- The package is APF partial-compiled against the lowest supported Angular version. Do not
  raise that floor without updating the peer range and the README compatibility note.

## Verify before proposing a change

```sh
npx tsc --noEmit
bun test/smoke.ts
bun test/jsnq-optional.test.ts
npm run build         # ng-packagr
npm pack --dry-run
```

All four must pass. `bun install` prints a `404 @adsq/jsnq` line for the unpublished peer
in some setups; that is expected and not a failure — check the exit status instead.
